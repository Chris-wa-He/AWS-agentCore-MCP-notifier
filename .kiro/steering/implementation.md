# 代码实现规范

## 核心原则

**在实现任何代码之前，必须确保使用的是最佳实践和最新的官方推荐方式。**

## 强制要求

### 文档优先原则

⚠️ **严禁仅凭训练数据或经验进行代码实现**

当遇到以下情况时，**必须先查询官方文档**：

1. **不确定 API 用法** - 查询官方 API 文档确认参数、返回值和使用方式
2. **不确定最佳实践** - 查询官方指南或推荐的实现模式
3. **使用第三方库** - 查询该库的官方文档确认正确用法
4. **版本相关特性** - 确认特性在目标版本中是否可用
5. **安全相关实现** - 必须查询最新的安全最佳实践
6. **性能敏感代码** - 查询官方性能优化建议

### 查询文档的方式

1. **使用搜索工具** - 搜索官方文档和可靠技术资源
2. **获取最新信息** - 使用网络工具获取最新的文档内容
3. **验证信息来源** - 优先使用官方文档，其次是权威技术博客

### 必须查询文档的场景

```
✅ 正确做法：
- "我需要实现 X 功能，让我先查询一下官方文档确认最佳实现方式..."
- "这个 API 的用法我不太确定，先查询文档验证一下..."
- "这个库的最新版本可能有变化，让我确认一下当前推荐的用法..."

❌ 错误做法：
- "根据我的经验，这样实现应该可以..."
- "我记得这个 API 是这样用的..."
- "通常情况下这样写就行..."
```

## 文档查询清单

在实现代码前，确认以下问题：

- [ ] 这个 API/库的官方推荐用法是什么？
- [ ] 是否有已知的陷阱或注意事项？
- [ ] 当前版本是否有特殊要求？
- [ ] 是否有更好的替代方案？
- [ ] 安全性方面有什么需要注意的？

## 适用范围

此规范适用于：

- 所有新功能的实现
- 现有代码的重构
- 第三方库的集成
- 配置文件的编写
- 部署脚本的编写

## 例外情况

以下情况可以不查询文档：

- 基础语法（如 Python 的 for 循环、if 语句等）
- 项目内部已定义的接口和函数
- 用户明确指定了实现方式

## 重要提醒

💡 **宁可多查一次文档，也不要凭记忆实现可能过时或错误的代码**

文档查询不是浪费时间，而是确保代码质量的必要步骤。
